820 //Order of operations:
821 // --
822 // SETUP : Sets up the scene ready for drawing.
823 // --
824 // UTILITY ID DRAW : Draws out utility textures from the scene. Currently this is CHARACTER IDS and TERRAIN WORLD POSITION.
825 // This can all be queried to find what the user is mousing-over.
826 // RAY DIR : For some of the shading stages, I cast rays out into the world. Unprojecting from the viewport to the world is
827 // pretty tedious, and pretty costly in a fragment shader. In an effort to lower the overhead, I draw the normals
828 // of a sphere to a texture to be queried later (which gives nearly the same result as projection). Although
829 // texture sampling can be expensive my hope is that it is cheaper than doing matrix multiplications on every
830 // fragment on screen a couple of times per frame.
831 // SHADOW PASS : This section draws shadow maps for the sun/moon. These maps are later referenced when lighting is applied.
832 // REFLECTIONS : The scene is drawn upside down, to be used for reflections later on. This is accomplished by scaling the camera by -1 in the y
833 // then drawing as usual.
834 // DISPLACEMENT : The textures that drive the water are generated. This is done in two passes. First, the displacement map is created,
835 // then this texture is fed into another draw call to generate a normal map.
836 // --
837 // RAW DATA PASS : The heart of the deferred rendering pipeline, the scene is drawn to a set of buffers, to which lighting is later applied.
838 // SKY : The sky is drawn to the back buffer.
839 // LIGHTING : Lighting is applied to our data from RAW DATA PASS.
840 // --
841 // DEFERRED BLUR PASS : The first of the post-processing stages, the lit scene from the LIGHTING stage is run through a blur shader.
842 // FORWARD SHADING : Forward-shaded elements are drawn. Currently this is only the water, which is drawn into the post effects framebuffer,
843 // ready for a DOF pass.
844 // DRAW BOX WHERE MOUSE IS : Inelegantly named, this stage draws a blue box where the mouse is. On Felix's suggestion, there are several
845 // conditions to it being drawn, such as the mouse being over the world/not clicking and dragging.
846 // FORWARD BLUR PASS : Blurs forwarded shaded elements, in the same way that DEFERRED BLUR PASS operates.
847 // BUTTONS : Draws the GUI to the screen after everything else has completed.
848 // DEBUG DRAW : Originally used to debug the CSM shadowing I was using, this is now largely obsolete. Should be removed before submission,
849 // but for now it can be used to draw points stored in m_debugPoints.
